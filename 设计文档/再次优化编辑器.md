# Role
你是一位精通 Tiptap 和 React 的高级前端工程师。

# Context
我们正在开发一个 Linear 风格的笔记应用。目前 Tiptap 编辑器已经运行，但“代码块”功能非常简陋：
1. 没有语法高亮颜色（可能是缺少 CSS）。
2. 无法复制代码。
3. 不知道当前是什么语言。

# Goal
请创建一个自定义的 React 组件（Node View）来替换默认的代码块渲染，并修复高亮显示问题。

# Requirements

## 1. 创建 `CodeBlockComponent.tsx` (Custom Node View)
使用 Tiptap 的 `NodeViewWrapper` 和 `NodeViewContent` 构建一个组件。
- **布局结构**:
  - 外层容器：`relative group rounded-lg overflow-hidden my-4`。
  - **顶部工具栏 (Toolbar)**:
    - 位于代码块顶部，背景色略深于代码背景 (`bg-gray-700` vs `bg-gray-800`)，或者悬浮显示。
    - **左侧**: 一个下拉菜单 (`<select>`) 或点击切换按钮，用于通过 `updateAttributes` 修改 `language` 属性。
    - **右侧**: 一个“复制”按钮。点击后将代码内容写入剪贴板，并将图标暂时变为“对号 (Check)”，2秒后恢复。
  - **代码区域**:
    - 使用 `<NodeViewContent as="code" />`。
    - 必须保留 `pre` 标签的语义结构以便 highlight.js 识别。

## 2. 视觉风格 (Linear Dark Mode)
- **配色**: 
  - 背景: `bg-[#1e1e1e]` (VS Code 风格深灰)。
  - 字体: `font-mono text-sm`。
  - 顶部工具栏: 只在 hover 时显示，或者一直显示但颜色低调。
- **复制交互**: 按钮要是无边框的 Icon (使用 Lucide `Copy` / `Check`)，颜色 `text-gray-400 hover:text-white`。

## 3. 修复语法高亮 (Critical)
- 明确指出在 `App.tsx` 或 `main.tsx` 中需要引入 Highlight.js 的 CSS 主题。
- 推荐使用 `import 'highlight.js/styles/atom-one-dark.css'` (或类似的高对比度暗色主题)。
- 确保 `CodeBlockLowlight` 扩展配置正确，并加载了 `common` 语言包 (js, ts, python, html, css 等)。

## 4. Tiptap 配置更新
- 提供更新后的 `Editor.tsx` 片段，展示如何在 `extensions` 数组中正确注册这个 React Component (`ReactNodeViewRenderer`)。

# Deliverables
1. `npm install` 命令（如果需要额外的 highlight.js 依赖）。
2. `CodeBlockComponent.tsx` 的完整代码。
3. `Editor.tsx` 中配置 `CodeBlockLowlight` 并绑定自定义组件的代码片段。
4. 提醒我在哪里引入 CSS 文件。
