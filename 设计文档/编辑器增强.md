# Role
Act as a Senior React Developer focused on Text Editor interactions.

# Context
We are upgrading the Editor component of our Linear-style note app.
Current Stack: React, Tailwind, Lucide Icons.
Goal: Implement **"Read vs Edit" modes** and **Syntax Highlighting Code Blocks**.

# Requirements

## 1. Tech Stack Additions
You need to use Tiptap's ecosystem.
- `@tiptap/react`
- `@tiptap/starter-kit`
- `@tiptap/extension-code-block-lowlight` (For syntax highlighting)
- `lowlight` (The core highlighting engine)

## 2. Feature: Read vs. Edit Mode
- **State:** Create a state `isEditing` (boolean, default `true`).
- **UI Control:** Add a Toggle Button in the top header (next to the date/breadcrumbs).
  - Icon: Use `Eye` (Lucide) for "View Mode" active, `PenLine` (Lucide) for "Edit Mode".
  - Style: Subtle, minimalist button (`text-gray-400 hover:text-gray-900`).
- **Behavior:**
  - Pass `editable={isEditing}` to the Tiptap editor instance.
  - When in **View Mode**: The cursor should be hidden, and content cannot be changed.
  - When in **Edit Mode**: Standard editing behavior.

## 3. Feature: Enhanced Code Blocks
- **Functionality:** Support generic code blocks with syntax highlighting.
- **Visual Style (Linear Style):**
  - The Code Block must contrast with the white page.
  - Background: `bg-[#1F2937]` (Gray-800/900).
  - Text Color: `text-gray-100` (Light).
  - Font: `font-mono text-sm`.
  - Radius: `rounded-lg`.
  - Padding: `p-4`.
  - Margins: `my-4`.
- **Implementation:**
  - Use Tailwind's `[&_pre]:` selectors within the parent wrapper to target the `pre` and `code` tags generated by Tiptap.
  - OR use a custom `CodeBlockComponent` logic if simpler for Tiptap integration.

## 4. Visual Refinements (Polishing)
- **Title Area:**
  - In **Edit Mode**: Display the `<input>` for the title.
  - In **View Mode**: Display an `<h1 className="text-4xl font-bold mb-4">` instead of the input, to look more like a document.
- **Typography:** Ensure `prose` classes are optimized so the content looks professional (Inter font, careful line-height).

# Output
1. Provide the **npm install** command for the new packages.
2. Provide the updated `Editor.tsx` code.
3. Include the logic for importing CSS for syntax highlighting (e.g., `highlight.js/styles/github-dark.css` or similar).

# Key Detail
Make sure the transition between Edit and View is instant (React state change), not a page reload.
